═══════════════════════════════════════════════════════════════
SECTION 1 — WHAT THIS SYSTEM IS
═══════════════════════════════════════════════════════════════

FinQuant is an automated trading decision agent. It watches 15-minute
SOL/USD perpetual futures price bars and decides for each bar:
  - ENTER_LONG  (buy — expects price to rise)
  - ENTER_SHORT (sell — expects price to fall)
  - NO_TRADE    (stand down — insufficient edge)
  - INSUFFICIENT_DATA (too ambiguous to decide)

It does NOT execute trades. Every decision is logged with full reasoning.

The decisions in this package are from a LIVE Pyth oracle data feed —
real SOL-PERP prices, April–May 2026.

═══════════════════════════════════════════════════════════════
SECTION 2 — THE PRIME DIRECTIVE (rules the agent must follow)
═══════════════════════════════════════════════════════════════

P-1 NEVER LIE. If it does not know, it must output INSUFFICIENT_DATA.
    Fabricated confidence is worse than no answer.

P-2 REASON WITH TOOLS. Every thesis must cite specific numeric values:
    RSI, ATR%, regime. "Price looks strong" without numbers = violation.

P-3 RISK-AVERSE DEFAULT. Default is NO_TRADE. Entries require confluence
    across multiple independent signals.

P-4 PATTERN SIMILARITY. Decisions should be anchored in prior validated
    patterns when memory is available.

P-5 CONTEXT FIRST. Read the regime and RSI trajectory before looking at
    the current bar value. RSI=58 in bar 3 of a fresh trend ≠ RSI=58 in
    bar 18 of an exhausted trend.

P-6 LONG-RUN MATH. Optimize dollars won minus dollars lost. A strategy
    can have 35% win rate and be profitable if wins are large enough vs losses.

═══════════════════════════════════════════════════════════════
SECTION 3 — THE HARD RULES (R-002 and R-003)
═══════════════════════════════════════════════════════════════

R-002 — CONFIDENCE SPREAD GATE:
  The agent must form two competing hypotheses (H1 and H2) with confidence
  values between 0.0 and 1.0.
  Confidence spread = H1_confidence - H2_confidence.
  If spread < 0.20 AND action is ENTER_LONG or ENTER_SHORT → VIOLATION.
  The agent must output NO_TRADE or INSUFFICIENT_DATA instead.

R-003 — RISK/REWARD GATE:
  Every entry must have a defined stop and target.
  Stop = 1.6 × ATR14 from entry price.
  Target = 4.0 × ATR14 from entry price.
  R-multiple = 4.0 / 1.6 = 2.5
  Breakeven win rate at R=2.5 = 1 / (1+2.5) = 28.57%
  Entries with R < 1.5 should be rejected.

═══════════════════════════════════════════════════════════════
SECTION 4 — HOW TO INTERPRET OUTCOMES
═══════════════════════════════════════════════════════════════

"win"              = Entry trade made money. Good decision.
"loss"             = Entry trade lost money. Bad decision.
"no_trade_correct" = Agent stood down; market did not make a significant
                     move. Good decision (correctly abstained).
"no_trade_missed"  = Agent stood down; market moved significantly.
                     Bad decision (missed opportunity).

THE CORRECT SUCCESS METRIC is:
  Decision Quality = (wins + no_trade_correct) / total_opportunities

This measures good decisions across ALL opportunities, not just win rate
on entries taken. At random, this should be ~50%.

═══════════════════════════════════════════════════════════════
SECTION 5 — RSI DIVERGENCE (the primary entry signal)
═══════════════════════════════════════════════════════════════

Bullish divergence (supports ENTER_LONG):
  Price makes a LOWER LOW while RSI makes a HIGHER LOW.
  Sellers are losing strength. Momentum shifting upward.

Bearish divergence (supports ENTER_SHORT):
  Price makes a HIGHER HIGH while RSI makes a LOWER HIGH.
  Buyers are losing strength. Momentum shifting downward.

No divergence = lower conviction. Agent should require more confluencing
signals or output NO_TRADE.

How to operate BLACK BOX today

Source basis: docs/blackbox_system_usage_current.md

This topic is the short operator read: what exists now, which surfaces are authoritative, and how to think about the current layered interaction model.

Current operating shape

  • BLACK BOX is a layered system, not a single UI.
  • Portal/UI is the human-facing control surface.
  • Engine/runtime is the source of truth behind it.
  • Anna is the analyst/strategy surface, not a live executor by herself.
  • Billy / exchange path is the execution connector when that lane is present.

What operators should use

Use the portal for structured access, navigation, and reading surfaces. Use governed runtime and API-backed controls when they are live. Do not treat scattered chat commands or raw logs as the primary product interface if a clearer portal surface exists.

What is authoritative today

Authoritative surfaces

  • The engine/runtime path for truth.
  • Portal route/role boundaries for user interaction.
  • Governed docs and plans for declared behavior.

The portal should show honest placeholders until those authoritative engine paths are wired. It should not fake a healthy or complete system state.

Web architecture and portal contract

Source basis: UIUX.Web/WEB_ARCHITECTURE_CANONICAL.md

This topic is the short operator/developer read on what the web layer is supposed to be, how it should be structured, and how it should talk to the engine.

What the web layer is

  • A stable front door and operational portal over the engine core.
  • Static-first where possible, with minimal JavaScript where interaction requires it.
  • A controlled route structure separating landing, login, internal portal, and external or consumer-safe surfaces.

What the web layer is not

  • Not an independent product brain.
  • Not a direct mount into engine internals.
  • Not a place to improvise routes or role semantics ad hoc.

Critical contracts

Binding contracts

  • Simple and stable route structure.
  • Separated internal and external portal surfaces by role.
  • Fail-closed behavior for unauthorized or degraded states.
  • Portal-to-engine integration through the API boundary only.
  • Reusable styling primitives instead of page-by-page invention.

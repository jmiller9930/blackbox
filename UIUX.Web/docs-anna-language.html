How to interact with Anna using natural language

Source basis: UIUX.Web/guide.html, docs/blackbox_system_usage_current.md, docs/runtime/execution_context.md

Anna is conversation-first. People should be able to ask in normal language, challenge a thesis, request clarification, or suggest training without needing command syntax first.

Default interaction model

Start with plain language. Ask what you are seeing, what Anna thinks, what risk is implied, or what would invalidate a view. The goal is not to memorize commands. The goal is to keep Anna usable as an analyst surface under normal conversation.

  • Ask for an explanation in ordinary terms.
  • Ask for the thesis, not just the answer.
  • Challenge weak logic directly if something does not feel supported.
  • Ask for status or review when you want operating-state clarity.

Useful topic patterns

Natural-language topics

  • Explanation request — “Explain why this setup looks strong.”
  • Challenge — “What would make this thesis wrong?”
  • Counter-argument — “What is the bearish case here?”
  • Status request — “What is your current read?”
  • Review request — “Review this setup and tell me the weak points.”
  • Training suggestion — “This looks like something Anna should learn.”

Conversation is not the same as training

Casual dialogue should stay conversational until explicitly staged. Anna should not silently mutate behavior from ordinary chat. If something sounds training-relevant, the system should classify it, explain why it matters, and ask whether it should be staged.

Compact training packet

  • classification
  • why
  • recommended_next_action
  • confirm?

What good Anna interaction should feel like

Anna should be responsive, readable, and challengeable. If she is uncertain, that uncertainty should be explicit. If she has a thesis, it should be inspectable. If you disagree, you should be able to push back in natural language without dropping into system-internal syntax.

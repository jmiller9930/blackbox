Why the web UI exists

Source basis: UIUX.Web/WEB_UI_REQUIRED_CONTEXT.md

The web UI is not optional polish. It is the required portal layer that makes BLACK BOX usable to operators and users as a governed system.

Core statement

BLACK BOX needs a stable human-facing portal for access, routing, visibility, control, onboarding, recovery, and governed interaction with the engine. Without that layer, key workflows become hidden, awkward, or unusable.

What the UI solves

  • A recognizable front door to the platform.
  • Internal operator visibility and control surfaces outside Slack.
  • Consumer-safe or external-safe portal flows.
  • PiBot pairing, recovery, and edge lifecycle surfaces.
  • Separation between portal, engine, and edge device responsibilities.

Why Slack is not enough

Slack can stay useful for operator interaction, but it is not the branded front door, not the right place for ordinary onboarding, not the right place for stable role-based navigation, and not the right place for long-lived portal identity.
